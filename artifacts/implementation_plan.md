# خطة تحويل الصفحة الثابتة إلى ديناميكية وبناء المحرر المرئي (Visual Builder Plan)

بناءً على طلبك ومراجعتي لملف `challet.html` الثابت (والذي يحتوي على أقسام رائعة مثل Hero، Story، Facilities، و Location)، وإضافاتك الجديدة لقاعدة البيانات، إليك التقرير والخطة الشاملة لتحويلها إلى صفحة ديناميكية بالكامل قابلة للتعديل من لوحة التحكم، مع الإجابة على سؤالك حول "المحرر المرئي المشابه لووردبريس".

## 1. الإجابة على سؤالك: هل يمكن بناء محرر مرئي يشبه ووردبريس؟
**نعم، بالتأكيد!** 
في معمارية النظام الحالية (React 4-Layer)، لدينا خياران لبناء ذلك:
1. **الخيار الأول (Live Preview - الأنسب لمشروعنا):** تقسيم شاشة الأدمن إلى نصفين؛ النصف الأيمن يعرض الصفحة الحية (Live Render)، والنصف الأيسر يحتوي على حقول التعديل (النصوص، الصور، المكعبات). بمجرد تعديل الحقل في اليسار، تتحدث الصفحة في اليمين لحظياً بدون حفظ.
2. **الخيار الثاني (Drag & Drop Builder مثل Elementor):** استخدام مكتبات مثل `Craft.js` أو `GrapesJS` داخل لوحة التحكم للسماح للأدمن بسحب وإفلات الأقسام وتعديل النصوص بالنقر المباشر عليها (Inline Editing). هذا الخيار ممكن جداً ولكنه يتطلب وقتاً أطول للبرمجة وتعديلاً جذرياً للمكونات.

---

## 2. تقرير: ما الذي يجب تغييره في `challet.html`؟
الملف الحالي هو ملف HTML ثابت موجود في مجلد `public/`. لكي يصبح ديناميكياً، يجب:
- **تفكيكه:** لا يجب أن يبقى كملف HTML. يجب تحويله إلى مكونات React منفصلة (مثال: `HeroSection.jsx`، `StorySection.jsx`، `FacilitiesSection.jsx`) ليتمكن من استهلاك البيانات بشكل احترافي.
- **ربطه بالـ Backend:** النصوص الحالية، الأرقام (مثل التقييمات 4.9)، والصور يجب أن تُقرأ من قاعدة البيانات عبر استدعاء API (تحديداً مسار مخصص للصفحة الرئيسية).

---

## 3. خطة التنفيذ (Implementation Plan)

### المرحلة الأولى: تجهيز قاعدة البيانات (Database Mapping)
بما أنك أضفت معلومات كثيرة للـ Database، سنقوم بضبط هيكل لحفظ محتوى الصفحة الرئيسية بالكامل داخل جدول `Client` (أو جدول جديد يسمى `PageContent`) كنظام `Jsonb`:
- `hero_section`: يحتوي على (العنوان الرئيسي، العنوان الفرعي، صورة/فيديو الخلفية).
- `story_section`: يحتوي على (نصوص القصة، الأرقام والإحصائيات المعروضة).
- `facilities_section`: مصفوفة من (اسم المرفق، الأيقونة، الوصف).
- `location_section`: يحتوي على (نص الموقع، رابط الخريطة).

### المرحلة الثانية: بناء مكونات React الديناميكية (Frontend Reactification)
- تحويل `challet.html` إلى صفحة رئيسية في مسار `src/pages/smar/spatial/ChalletPage.jsx`.
- استبدال كل نص ثابت (Hardcoded) بمتغير يأتي من استدعاء البيانات (مثال: `{config.hero_title}` بدلاً من النص الثابت).
- الحفاظ على كل حركات الـ CSS المذهلة والـ Animations دون تغيير عن طريق نقل الـ CSS إلى ملفات `module.css` أو استخدام Tailwind بشكل مدروس.

### المرحلة الثالثة: بناء "المحرر المرئي" في لوحة التحكم (Admin Visual Editor)
- إنشاء تبويب جديد في الـ Admin Dashboard يسمى "منشئ الصفحات" أو "المحرر المرئي" (Visual Builder).
- **تصميم الواجهة:**
  - عرض إطار (iframe) على يمين الشاشة يعرض صفحة `ChalletPage.jsx` الحية.
  - على يسار الشاشة، عرض قائمة بالأقسام (Hero, Story, Facilities).
  - عند النقر على أي قسم، تظهر حقول التعديل. عند كتابة حرف جديد، يتم إرسال إشارة (postMessage) للـ iframe لتحديث النص أمام عينيك فوراً (Live Binding).
- ربط زر "حفظ" (Publish) بحفظ هذا الـ JSON في قاعدة البيانات ليعكس للزوار.
