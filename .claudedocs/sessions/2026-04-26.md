## 1. ما الذي تغيّر اليوم

### 1A — خطأ اكتُشف
```
POST /api/v1/public/smar/auth/register → 404
```
هذا المسار **لم يُبنَ** — إنما جاء من طلب يدوي بمسار خاطئ.
الـ endpoint الحالي صحيح: `POST /api/v1/public/register`
لكن قُرِّر نقله إلى `/api/v1/auth/register` (راجع 1B).

### 1B — قرار معماري: نقل Registration من Public إلى Auth
**السبب:**
- التسجيل يُصدر JWT + يحط HttpOnly cookie → هو `auth` operation وليس `public`
- `auth.salmansaas.com` subdomain موجود ومخصص لهذا
- Registration يجب أن يمر عبر نفس الـ SSO portal (SSOLoginPage)

**الوضع قبل:**
```
salmansaas.com/register → RegistrationPage → POST /api/v1/public/register
```

**الوضع المطلوب:**
```
salmansaas.com → "Start for Free" → auth.salmansaas.com/register
auth.salmansaas.com/register → Registration form → POST /api/v1/auth/register
                             → JWT + cookie + redirect → {slug}.salmansaas.com/dashboard
```

### 1C — قرار تجاري: نموذج Managed Trial (ليس Self-Service كامل)
أنت **لست Vercel أو Shopify** — أنت مزوّد خدمة managed.

```
الزبون يسجّل → يحصل على ديمو 14 يوم → أنت تراجع وتقرر:
  ✅ تفعيل وبناء صفحته الكاملة
  ➕ إضافة خدمات إضافية بناءً على طلبه
  ❌ رفض (الحساب ينتهي تلقائياً)
```

### 1D — قرار: فصل الخدمات
الشووكيس الحالي = **خدمة العقارات والحجز** (Real Estate)
مستقبلاً: خدمة مطاعم، خدمة فنادق، خدمة رياضة — كل منها له showcase منفصل.

## 3. التعديلات المطلوبة — مرتّبة بالأولوية

### 🔴 Priority 1 — Prisma Schema Migration

```prisma
model Client {
  // ...existing fields...

  // ── Trial & Lifecycle ────────────────────────────────────
  status         String    @default("trial")       @map("status")
  trial_ends_at  DateTime? @map("trial_ends_at")   @db.Timestamptz(6)
  service_type   String?   @default("real_estate") @map("service_type")
  notes          String?   @map("notes")
}
```

**Migration SQL:**
```sql
ALTER TABLE clients
  ADD COLUMN status        VARCHAR(20)  NOT NULL DEFAULT 'trial',
  ADD COLUMN trial_ends_at TIMESTAMPTZ,
  ADD COLUMN service_type  VARCHAR(50)  DEFAULT 'real_estate',
  ADD COLUMN notes         TEXT;

-- العميل الحالي يصير demo مش trial
UPDATE clients SET status = 'demo' WHERE slug = 'smar';
```

**Status values:**
| القيمة | المعنى |
|---|---|
| `trial` | سجّل حديثاً — في فترة 14 يوم |
| `active` | أنت فعّلته يدوياً |
| `suspended` | أنت أوقفته (ديون / مخالفة) |
| `expired` | انتهت التجربة ولم تُفعَّل |
| `demo` | بيئة تجريبية داخلية (مثل smar) |

### 🔴 Priority 2 — Backend: نقل Registration إلى Auth

**الملفات:**

#### `app/api/v1/admin/auth.py` — إضافة endpoint جديد
```python
# POST /api/v1/auth/register
class TenantRegistrationRequest(BaseModel):
    owner_name:        str
    email:             EmailStr
    password:          str           # min 8
    business_name_ar:  str           # → Client.name_ar + Client.name
    business_name_en:  str | None    # → Client.name_en
    whatsapp_number:   str           # → Client.phone + whatsapp_number
    venue_type:        str           # real_estate | restaurant | hotel | sports
    currency:          str = "USD"   # default USD للبنان
    payment_methods:   list[str] = ["cash", "card"]
    primary_color:     str = "#6d28d9"
```

**Logic داخل الـ route:**
```
1. يستدعي registration_service.register_new_tenant()
2. بعد الإنشاء: يستدعي create_access_token() مثل client_login
3. يحط HttpOnly cookie على .salmansaas.com (مثل client_login بالضبط)
4. يرجع { token, slug, trial_ends_at, dashboard_url }
```

#### `app/services/registration_service.py` — تحديثات
```python
# إضافة venue_type → unit_types mapping
VENUE_TYPE_MAP = {
    "real_estate": ["villa", "chalet", "pool", "restaurant"],
    "restaurant":  ["restaurant"],
    "hotel":       ["room", "suite", "villa"],
    "sports":      ["court", "field", "gym"],
}

# إضافة trial logic
from datetime import datetime, timedelta, timezone

trial_ends_at = datetime.now(timezone.utc) + timedelta(days=14)

# إنشاء Client مع status + trial_ends_at
client = await repo.create_client({
    ...
    "status":        "trial",
    "trial_ends_at": trial_ends_at,
    "service_type":  data["venue_type"],
    "currency":      data.get("currency", "USD"),
    "unit_types":    VENUE_TYPE_MAP.get(data["venue_type"], ["chalet"]),
    "payment_methods": data.get("payment_methods", ["cash", "card"]),
})
```
