## 🔧 Technical Decisions Made

- **Relume via npm لا MCP:** MCP يحتاج auth تفاعلي — استخدمنا npm package مباشرة (`@relume_io/relume-ui`) بدل انتظار الـ auth
- **`legacy-peer-deps=true` في `.npmrc`:** React 19 vs Relume's React 18 peer dep — الـ flag دائم في الملف بدل تكراره مع كل install
- **"الكل" category:** API ما عنده endpoint واحد للكل — نجلب `/restaurant/menu` كاملاً ونفلتر client-side
- **hookCategoryId guard:** `const hookCategoryId = activeCategoryId === '__all__' ? null : activeCategoryId` — يمنع الـ hook من استدعاء endpoint غير صحيح
- **Arizona ≠ Caracas:** قررنا تمييز واضح — Arizona (fast-casual، مرح، ساطع) vs Caracas (fine dining، سينمائي، داكن) — لعرضهم على العملاء كـ demos مختلفين
