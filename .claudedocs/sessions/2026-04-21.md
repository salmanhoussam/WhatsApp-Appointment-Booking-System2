## Priority 1 — Phase 36 State Sync Fix ✅ DONE

**Problem:**
After uploading or deleting an image in `UnitFormModal`, the parent `SmarAdminDashboard`'s `units` array stayed stale. Re-opening the modal showed the old image list (no newly uploaded images).

**Root cause:**
`handleUpload` and `handleDelete` in `UnitFormModal.jsx` only called `setImages(data.images)` (local modal state), never notified the parent component.

**Fix — 3 changes:**

**`frontend/src/pages/smar/admin/UnitFormModal.jsx`**
- Function signature: added `onImagesChange` prop
  ```jsx
  export default function UnitFormModal({ isOpen, onClose, unit, onSave, onImagesChange })
  ```
- `handleUpload` success: added `onImagesChange?.(data.images || [])`
- `handleDelete` success: added `onImagesChange?.(data.images || [])`

**`frontend/src/pages/smar/admin/SmarAdminDashboard.jsx`**
- Wired the `onImagesChange` prop at `<UnitFormModal>` (line ~1595):
  ```jsx
  onImagesChange={(imgs) =>
    setUnits(us => us.map(u =>
      u.id === selectedUnit?.id
        ? { ...u, images: imgs, image_url: imgs[0] ?? null }
        : u
    ))
  }
  ```
- Optimistic update — no full refetch needed. Parent `units` array stays in sync with every upload/delete.

## Priority 2 — Phase 43: Dynamic Content Block Builder ✅ DONE

### Background
`database_report.md` (written 2026-04-20) identified 3 critical gaps in the Unit model:
- ❌ No `category` field (separate from `unit_type`)
- ❌ No bilingual description (`description_ar` / `description_en`)
- ❌ No dynamic JSON fields for amenities, content, and rules/policies — everything hardcoded in the frontend

`خطة المحتوى الديناميكي.md` (written 2026-04-20) defined the solution: a Headless CMS-style Block Builder with JSONB fields and a tri-level content hierarchy.

---

### 2.1 — Schema Migration (`prisma/schema.prisma`) ✅

Added 6 new fields to the `Unit` model:

```prisma
// ── Dynamic Content Block Builder ─────────────────────────────────────────
category       String? @map("category")        // villa | chalet | studio — filterable classification
description_ar String? @map("description_ar")  // Arabic description (replaces single 'description')
description_en String? @map("description_en")  // English description

// Block Builder JSON — array of {type, content, style?, icon?, title?}
// Types: section_title, highlight_item, paragraph
content_blocks Json? @map("content_blocks")

// Amenities — array of {icon, label, label_ar?}
amenities Json? @map("amenities")

// Rules & Policies — {checkIn, checkOut, cancellation, rules[]}
rules_policies Json? @map("rules_policies")
```

All fields are nullable — no migration breaking change. Existing units return `null` for new fields.

**DB push command used:**
```bash
npx prisma db push   # via DIRECT_URL (port 5432)
prisma generate      # regenerate prisma-client-py
```

### 2.5 — Admin UI: UnitFormModal Rewrite (`frontend/src/pages/smar/admin/UnitFormModal.jsx`) ✅

The modal was completely rewritten with 3 tabs:

#### Tab 1 — التفاصيل (Details)
Fields: name AR/EN, category dropdown (شاليه/فيلا/ستوديو/مطعم/مسبح), capacity, bedrooms, bathrooms, price, price_label.
Category uses emoji-labeled select (`<select>` styled inline).

#### Tab 2 — الصور (Images)
- 3-col thumbnail grid with trash-icon hover overlay per image
- Dropzone: dashed gold border (`rgba(212,168,83,0.35)`), drag-and-drop + click-to-browse
- Spinner on upload in progress
- Upload: `POST /units/{unit.id}/images` with FormData
- Delete: `DELETE /units/{unit.id}/images` with `{ data: { url } }`
- Only shown for existing units (`unit?.id` required)
- State synced to parent via `onImagesChange` prop (Phase 36 fix above)

#### Tab 3 — المحتوى (Content Block Builder)
Three sub-editors in vertical stack:

**1. ContentBlockEditor** — block array builder
- Add buttons: "عنوان رئيسي" (gold) / "ميزة بارزة" (green) / "نص وصفي" (blue)
- Each block renders an inline editor card with move-up/move-down/delete controls
- `section_title`: content textarea + style options (size: small/medium/large, color: gold/white/muted, bold toggle)
- `highlight_item`: icon text input + title + content
- `paragraph`: textarea + style options

**2. AmenitiesEditor** — amenities array
- `+` add button creates `{icon, label}` pair
- Icon field shows icon name (from predefined list: wifi, pool, mountain, car, flame, snowflake, tv, utensils, bath, bed-double, tree-pine, sun, moon, coffee, music, shield, sparkles, star, heart, baby, wind, dumbbell, parking-circle)
- Label text input
- Delete button per row

**3. RulesEditor** — rules_policies object
- Check-in time input (HH:MM)
- Check-out time input (HH:MM)
- Cancellation policy textarea
- Rules list — add/remove individual rule strings

#### Save behavior
- Tab 1 save: `PATCH /units/{id}` with details fields
- Tab 3 save: `PATCH /units/{id}` with `{ content_blocks, amenities, rules_policies }`
- Tab 2: saves immediately on upload/delete (no save button)

**File:** `scripts/monitor_health.py`

**Usage:**
```bash

```

**Cron (every 20 min):**
```

```

**Output:**
- `✅ [timestamp] System Health: OK. Login successful. Latency: 342ms`

**Exit codes:** 0 = OK, 1 = FAIL (cron-compatible).
