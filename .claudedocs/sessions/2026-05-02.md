### 46.9 — app/api/v1/public/store.py (جديد)
Public store endpoints (no auth, gated by require_service("store")):
```
GET  /api/v1/public/store/products                          → catalog (category, brand, featured, search)
GET  /api/v1/public/store/products/{id}                     → product details + category + brand
GET  /api/v1/public/store/categories                        → top-level categories (parentId=None)
POST /api/v1/public/store/cart                              → add to cart (session UUID, 7-day TTL)
GET  /api/v1/public/store/cart/{session_id}                 → view cart with product details
DELETE /api/v1/public/store/cart/{session_id}/items/{id}    → remove item
POST /api/v1/public/store/orders                            → checkout (clears cart after success)
GET  /api/v1/public/store/orders/{order_id}                 → order status tracking
```
Cart design: يعتمد على `sessionId` UUID من localStorage — لا login مطلوب.
Total يُحسب دائماً من DB prices (لا ثقة بالـ frontend).

### 46.10 — app/api/v1/admin/store.py (جديد)
Admin store endpoints (JWT required + require_service("store")):
```
GET/POST /admin/store/products                     → list + create
PATCH/DELETE /admin/store/products/{id}            → update + delete
GET/POST /admin/store/categories                   → list + create (with parentId hierarchy)
PATCH/DELETE /admin/store/categories/{id}
GET/POST /admin/store/brands                       → list + create
PATCH/DELETE /admin/store/brands/{id}
GET /admin/store/orders                            → list with status filter
PATCH /admin/store/orders/{id}/status              → pending|processing|shipped|delivered|cancelled|refunded
GET /admin/store/orders/stats                      → today's revenue + counts by status
```
Product names stored as JSON `{"ar": "...", "en": "..."}` — multilingual support.
