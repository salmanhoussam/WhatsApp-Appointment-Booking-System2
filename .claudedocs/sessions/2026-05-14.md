## 🔧 Technical Decisions Made

- **`page_type` is a config field, not a route** — `"showcase"` renders ConfigurableHero as fallback; `"catalog"` redirects to `./catalog`; sections always win over page_type when configured.
- **`publicApi` over raw `axios`** — DynamicPage now reads `VITE_API_URL` via the shared `publicApi.js` instance. One env var controls all API calls for production.
- **`_dynamic.routes.jsx` fallback** — Unregistered tenants get DynamicPage + CatalogPage + CartPage automatically. No tenantRegistry entry needed for pipeline-created tenants.
- **postMessage bridge** — Dashboard sends `{ accent, heroType, catalogLayout }` → DynamicPage maps to `{ primary_color, page_type, config.catalog_layout }` for live preview without page reload.
- **`GalleryEditor.handleFiles` functional updater fix** — Using local `let images` accumulator instead of `onChange(prev => ...)` since `onChange` accepts a data object, not a function.
