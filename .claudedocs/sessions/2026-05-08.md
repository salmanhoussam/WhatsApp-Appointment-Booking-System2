## 🔧 Technical Decisions Made

- **Prisma Python quirk**: تمرير `None` صريح لـ `Json?` field يرفع `MissingRequiredValueError` — الحل: إما حذف الـ key أو تغليفه بـ `Json(value)`. وُثِّق في catalog.py + services.py
- **Seed على localhost لا production**: الكود الجديد (catalog routes) لم يُرفع لـ Railway بعد، الـ seed يشتغل على السيرفر المحلي الذي عنده الكود الحديث
- **`module_key` يمشي من Template → Request → DB**: `template-registry.js` → `seed-from-template` body → `CatalogCategory.moduleKey` — سلسلة متكاملة الآن
- **`venue_type` mapping**: `{ store: 'ecommerce', restaurant: 'restaurant', catalog: 'services' }` في `TenantRegisterPage` — يحدد `service_type` في DB
- **`useImageUpload`**: hook يعيد `{ upload, uploading, error, reset }` — `upload(file, { context, ...ids })` ترمي exception عند الفشل للـ caller يتعامل معها
- **OverviewTab catalog stats**: جلب `/catalog/categories` + `/catalog/items` مرة واحدة عند mount — الـ count يذهب لـ StatCards، أول 5 items يذهبون لـ TopCatalogItems (لا double fetch)
- **Image upload flow في CatalogTab**: للـ create — ينشئ الـ item أولاً ليحصل على UUID، ثم يرفع الصورة بـ `catalog_item` context، ثم يـ patch الـ image_url. للـ edit — يرفع مباشرة. إذا لم يُختر file → URL input كـ fallback
- **KanbanBoard مخفي لـ `catalog` module**: لا طلبات في الـ catalog-only tenants — البورد لا معنى له
- **Order stat cards مشروطة**: تظهر فقط لـ `restaurant`/`store` — الـ catalog tenant يرى بطاقتي الأقسام والمنتجات فقط
