# Session Plan — 22 April 2026
# Property JSON Ingestion Format — Schema Migration & Seed Script Update

---

## Context

The client delivers unit/property data in a structured JSON format (see sample below).  
The seed script and DB schema need to be updated to ingest this format without manual reshaping.

```json
{
  "property": {
    "name": "Zayin - Garden Cottage",
    "type": "Bed & Breakfast",
    "location": { "area": "Smar Jbeil", "region": "Batroun", "distance_info": "..." },
    "capacity": { "max_guests": 2, "bedrooms": 1, "bathrooms": 1, "area_sqm": null },
    "timings": { "check_in": "17:00", "check_out": "14:00", "quiet_hours_start": "00:00", "quiet_hours_end": "09:00" },
    "description": "...",
    "highlights": ["Luxurious Accommodation: ...", "Stunning Views: ..."],
    "amenities": ["Wifi", "Microwave", "Balcony", "AC", "Smart TV"],
    "facilities": ["Pool access (May 15 to Oct 5...)", "Pool Bar", "Dining / Clubhouse"],
    "services": { "included": ["Concierge..."], "additional_charge": ["Airport pick up..."] },
    "addons_and_pricing": {
      "sofa_bed":  { "price": 30, "currency": "USD", "unit": "per_person" },
      "crib":      { "price": 30, "currency": "USD", "unit": "per_stay" },
      "breakfast": { "status": "Optional", "time": "9:30 AM - 11:30 AM", "location": "Clubhouse" }
    },
    "policies": {
      "cancellation": ["100% refund up to 14 days...", "50% refund up to 7 days..."],
      "rules": ["Identity Cards required...", "Pets allowed..."]
    }
  }
}
```

## Phase 1 — Schema Migration (Prisma)

Add nullable fields to 3 models. All `@default` — zero breaking changes.

### `Property` model — add 3 fields:
```prisma
property_type  String?  @map("property_type")   // "Bed & Breakfast" | "Chalet" | "Villa"
location       Json?    @map("location")         // {area, region, distance_info}
facilities     Json?    @map("facilities")        // ["Pool access...", "Dining"]
```

### `Unit` model — add 1 field:
```prisma
area_sqm  Float?  @map("area_sqm")
```

### `Service` model — add 2 fields:
```prisma
is_included   Boolean  @default(false)  @map("is_included")    // free vs paid
pricing_unit  String?  @map("pricing_unit")  // "per_person" | "per_stay" | "per_night"
```

**Migration command:**
```bash
python -m prisma db push
python -m prisma generate
```

## Phase 2 — Normalization Layer (seed script)

Functions added to `scripts/seed_beit_smar_units.py`:

### `normalize_amenities(amenities_list)`
Converts `["Wifi", "AC", "Smart TV"]` → `[{"icon":"wifi","label":"Wifi"}, ...]`  
Uses `AMENITY_ICON_MAP` keyword dict for auto-icon selection. Fallback: `"star"`.

### `unit_from_property_json(data)`
Full converter: property JSON → unit dict compatible with UNITS array.  
- `highlights[]` → `content_blocks` (type: `highlight_item`)
- `timings` → `rules_policies.checkIn/Out/quietHoursStart/End`
- `policies.cancellation[]` → `rules_policies.cancellation` (joined)
- `policies.rules[]` → `rules_policies.rules`
- `facilities[]` → `rules_policies.facilities`
- `services.*` → `rules_policies.servicesIncluded/servicesChargeable`
- `location.distance_info` → appended as `paragraph` content_block
- `_addons` private key → passed to `seed_services_from_json()`

### `seed_services_from_json(db, client_id, prop_id, addons)`
Creates `Service` records from `addons_and_pricing`.  
- Skips if service with same `name_en` already exists (idempotent)
- Maps known addon keys to AR/EN name pairs
- Stores `pricing_unit` (after Phase 1 migration)

## Sprint — Cascade Delete Fix (22 Apr 2026)

### Problem
Deleting units from Supabase failed with a Foreign Key constraint error because the `Booking` table held references to those units with no cascade rule.

### Fix Applied — `prisma/schema.prisma` → `Booking` model

Added `onDelete: Cascade` to all three parent relations:

```prisma
client   Client   @relation(fields: [clientId],   references: [id], onDelete: Cascade)
customer Customer @relation(fields: [customerId], references: [id], onDelete: Cascade)
unit     Unit     @relation(fields: [unitId],     references: [id], onDelete: Cascade)
```

`BookingService` already had `onDelete: Cascade` on the `booking` side — no change needed there.

### Commands Run
```bash
npx prisma format          # ✅ formatted in 58ms
npx prisma db push         # ✅ synced to Supabase in 12.84s
python -m prisma generate  # ✅ Prisma Client Python v0.15.0 regenerated
```

### Result
- Deleting a `Unit` now automatically removes all its `Booking` rows.
- Deleting a `Client` now automatically removes all its `Booking` rows.
- Deleting a `Customer` now automatically removes all their `Booking` rows.
- Old test units can now be deleted from Supabase without constraint errors.
