## Priority 1 — useAdminRole backward compat ✅ DONE
**File:** `frontend/src/utils/useAdminRole.js`

**Problem:** Two token types break RBAC:
- Old JWT has `role: "manager"` (string before enum migration) → not in ROLE_TABS → all tabs blocked
- `Login.jsx` stores a CLIENT token (`type: "client"`, no `role` field) → `useAdminRole()` returns null → blocked

**Fix:** Map legacy and client tokens to TENANT_ADMIN:
```js
if (payload?.type === 'client') return 'TENANT_ADMIN';
const LEGACY = { manager: 'TENANT_ADMIN', admin: 'TENANT_ADMIN' };
return LEGACY[payload?.role] ?? payload?.role ?? null;
```

## Priority 2 — Fix Login.jsx endpoint ✅ DONE
**File:** `frontend/src/pages/admin/Login.jsx`

**Problem:** Calls `POST /api/v1/auth/login` (Client login → `type:"client"` token).
Staff members (User model) need `POST /api/v1/auth/users/login` (→ `type:"admin"` + `role` field).

**Fix:** Change endpoint to `/api/v1/auth/users/login`, update request body from
`{ identifier, password }` → `{ email, password }` (User login requires email not identifier).

## Priority 4 — SSOLoginPage dual-mode login ✅ DONE
**File:** `frontend/src/pages/auth/SSOLoginPage.jsx`

**Problem:** Only calls `/api/v1/auth/login` (Client login).
Staff members (MANAGER_RESERVATIONS, MANAGER_UNITS) are in the User table — they need
`/api/v1/auth/users/login` with email+password.

**Fix:** Try client login first. If 401, fallback to users/login.
Show appropriate error if both fail.

```js
// 1. Try client login
try { data = await axios.post('.../auth/login', { identifier, password }) }
// 2. If fail, try user login
catch { data = await axios.post('.../auth/users/login', { email: identifier, password }) }
```

## Priority 6 — Phase 36: Unit Image Upload (replace URL field) ✅ DONE

### Context
Current state in `UnitFormModal.jsx`: admin types a URL string manually into an input field.
Existing `upload.py` (`app/api/v1/admin/upload.py`) has a working Supabase Storage upload endpoint BUT:
- Uses a dummy hardcoded `get_current_client()` → returns `{"slug": "beitsmar"}` always
- No per-unit folder structure
- Not wired to real auth
- Not integrated with dashboard UI

### Target folder structure in Supabase Storage
```
properties/
  {slug}/
    units/
      {unit_id}/          ← one folder per unit (use unit UUID)
        {uuid}.jpg
        {uuid}.webp
        ...
```

Example live URL:
`https://wefjghagwpkotrrdiqyi.supabase.co/storage/v1/object/public/properties/smar/units/abc-123/img.jpg`

### Backend plan
**Fix `upload.py`:**
1. Remove dummy `get_current_client()` — replace with real `get_current_tenant()` from `app.core.tenant`
2. Add `unit_id` path parameter: `POST /api/v1/admin/units/{unit_id}/images`
3. File path: `{slug}/units/{unit_id}/{uuid}.{ext}`
4. After upload: update `Unit.images[]` array in DB (append new URL)
5. Add `DELETE /api/v1/admin/units/{unit_id}/images` to remove a URL from the array + delete from storage

**Architecture (4-layer):**
- Route → Service (`unit_image_service.py`) → Repository (`unit_repository.py`) → Supabase Storage + Prisma

### Frontend plan
**In `UnitFormModal.jsx`:**
- Replace the `image_url` text input with a **Dropzone component**:
  - Drag & drop or click to select (max 5 files, 5MB each, jpg/png/webp)
  - Preview thumbnails after selection
  - Upload on form save (or immediately on drop — TBD)
  - Show existing images as deletable thumbnail grid
  - GS MAR styling: dark surface, gold dashed border on hover
