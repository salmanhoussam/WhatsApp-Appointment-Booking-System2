### 🟠 Priority 3 — TenantRegisterPage لا تستخدم module_key بعد

صفحة التسجيل `frontend/src/pages/auth/TenantRegisterPage.jsx` تسيد الـ categories لكن **لا تمرر `module_key`** عند الـ seed.

**المطلوب:** في الخطوة 3 من التسجيل (seed-from-template):
```js
import { getServicesForTemplate, getModuleKey } from '@/config/template-registry';

const moduleKey = getModuleKey(templateId);       // 'restaurant' | 'store' | 'catalog'
const services  = getServicesForTemplate(templateId); // ['restaurant', 'reservations']

// عند seed الفئات:
await adminApi.post('/catalog/seed-from-template', {
  ...getSeedPayload(templateId),
  module_key: moduleKey,   // ← هذا الحقل ناقص
});

// عند تفعيل الخدمات:
await adminApi.post('/services/activate', { services }); // ← هذا endpoint لم يُبنَ بعد
```

- [ ] الـ sidebar يظهر مع الأيقونات والـ labels بالعربي

- [ ] `primary_color` من `/admin/settings` يُطبَّق على كل الـ UI

#### جلسة B — Orders Kanban
- [ ] 4 columns تظهر: معلق / يُحضَّر / جاهز / تم التسليم
- [ ] كل card تعرض: اسم العميل + عدد العناصر + المجموع + الوقت
- [ ] drag & drop يحرك الـ card ويحدّث الـ API (`PATCH status`)
- [ ] لو فشل الـ API → الـ card ترجع لمكانها (rollback)
- [ ] الـ module_key يُحدد: restaurant orders أو store orders (مش الاثنين معاً)

#### جلسة C — Activity Feed + Top Items
- [ ] الـ feed يعرض آخر 10 طلبات/حجوزات بـ timestamp relative ("منذ 5 دقائق")

- [ ] كل item تعرض: الاسم + عدد الطلبات + الإيراد
