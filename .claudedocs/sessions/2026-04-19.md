## Subdomain Routing Fix + Phase C: SSO тАФ COMPLETE (2026-04-19)

### Subdomain Routing Architecture (Permanent Lock)
- **Problem:** `smar.salmansaas.com/smar/showcase` тАФ slug duplicated in URL
- **Root cause:** App.jsx used `/:slug/*` for all modes; pathnameBase was `/smar` even on subdomain
- **Fix:**
  - `IS_SUBDOMAIN_MODE` constant in `App.jsx`
  - Subdomain: `<Route path="/*">`, Localhost: `<Route path="/:slug/*">`
  - Root redirect: `/` тЖТ `/showcase` on subdomain, `/smar` on localhost
  - `useTenantBase()` in `useTenantSlug.js`: `''` on subdomain, `/${slug}` on localhost
  - All navigate() calls in TenantHeader, ShowcaseTemplate, SpatialPropertyDetails, CollageScene, SmarBookingDrawer updated to use `${base}/...`
  - TenantResolver: legacy redirect strips `/${subdomain}` prefix
- **CORS hardcoded baseline:** `app/core/config.py` always allows smar/auth/salmansaas.com regardless of FRONTEND_URL env var
- **Docs:** `.claudedocs/new_tenant_checklist.md` тАФ 5-step checklist for every new tenant

## Phase D: RBAC Tab Guards тАФ COMPLETE (2026-04-19)

### Problem
All admin users saw ALL tabs regardless of role. A `MANAGER_RESERVATIONS` could access Settings and Team Management. No backend enforcement on sensitive write endpoints.

### DB Change
- `prisma/schema.prisma`: Added `enum UserRole { SUPER_ADMIN TENANT_ADMIN MANAGER_RESERVATIONS MANAGER_UNITS }` with `@@schema("public")`
- `User.role` migrated: `String @default("manager")` тЖТ `UserRole @default(TENANT_ADMIN)`
- Pushed to Supabase via `DATABASE_URL=DIRECT_URL npx prisma db push --skip-generate` (port 5432, bypasses pgbouncer)
- Python client regenerated: `python -m prisma generate`

### Frontend тАФ Role-Based Sidebar

**`src/utils/useAdminRole.js`** (NEW)
- `useAdminRole()` тАФ decodes JWT payload from `localStorage.admin_access_token` via `atob(token.split('.')[1])`, returns `role` string or `null`
- `ROLE_TABS` map тАФ defines allowed tabs per role:
  - `SUPER_ADMIN` / `TENANT_ADMIN`: all 9 tabs
  - `MANAGER_RESERVATIONS`: inbox, bookings, dashboard, housekeeping, maintenance, gardens
  - `MANAGER_UNITS`: units, dashboard, housekeeping, maintenance, gardens
- `canAccessTab(role, tabId)` тАФ returns boolean

**`SmarAdminDashboard.jsx`** (UPDATED)
- `useAdminRole()` called at root тАФ role passed as prop to `<Sidebar>`
- `Sidebar`: `navItems` filtered via `canAccessTab(role, item.id)` тАФ unauthorized tabs hidden entirely
- `renderTab()` replaces inline `{activeTab === 'x' && <XTab />}` pattern тАФ checks `canAccessTab` before render тЖТ returns `<UnauthorizedTab />` on violation
- `UnauthorizedTab` component: gold lock icon, "╪║┘К╪▒ ┘Е╪╡╪▒╪н ┘Д┘Г" Arabic message, GS MAR palette
- Default `activeTab` initialized to first tab in `ROLE_TABS[role]` (not hardcoded `'bookings'`)
- Top bar: role badge displayed (gold pill, `TENANT_ADMIN` format with underscores replaced by spaces)

### Backend тАФ Settings Write Guard

**`app/core/tenant.py`** (UPDATED)
- `require_roles(*allowed_roles)` dependency factory added
- Internally calls `get_current_admin_user()`, checks `user.role.value` against `allowed_roles`
- Raises `HTTP 403` with Arabic-friendly detail if role not permitted

**`app/api/v1/admin/settings.py`** (UPDATED)
- `PATCH /settings`: added `_user = Depends(require_roles("SUPER_ADMIN", "TENANT_ADMIN"))`
- `MANAGER_RESERVATIONS` and `MANAGER_UNITS` now get 403 on settings write
- GET `/settings` remains open to all valid admin tokens (read-only, safe)
