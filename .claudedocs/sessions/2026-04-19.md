## Subdomain Routing Fix + Phase C: SSO — COMPLETE (2026-04-19)

### Subdomain Routing Architecture (Permanent Lock)
- **Problem:** `smar.salmansaas.com/smar/showcase` — slug duplicated in URL
- **Root cause:** App.jsx used `/:slug/*` for all modes; pathnameBase was `/smar` even on subdomain
- **Fix:**
  - `IS_SUBDOMAIN_MODE` constant in `App.jsx`
  - Subdomain: `<Route path="/*">`, Localhost: `<Route path="/:slug/*">`
  - Root redirect: `/` → `/showcase` on subdomain, `/smar` on localhost
  - `useTenantBase()` in `useTenantSlug.js`: `''` on subdomain, `/${slug}` on localhost
  - All navigate() calls in TenantHeader, ShowcaseTemplate, SpatialPropertyDetails, CollageScene, SmarBookingDrawer updated to use `${base}/...`
  - TenantResolver: legacy redirect strips `/${subdomain}` prefix
- **CORS hardcoded baseline:** `app/core/config.py` always allows smar/auth/salmansaas.com regardless of FRONTEND_URL env var
- **Docs:** `.claudedocs/new_tenant_checklist.md` — 5-step checklist for every new tenant

### Phase C: SSO Subdomain
- `src/pages/auth/SSOLoginPage.jsx` — standalone enterprise login for `auth.salmansaas.com`
- Backend: `POST /api/v1/auth/login` + `POST /api/v1/auth/users/login` both set HttpOnly cookie on `.salmansaas.com`
- Token handoff via URL: `?token=JWT` → `ProtectedRoute.jsx` reads, saves to localStorage, removes from URL
- `TenantResolver.jsx`: `auth.*` hostname → renders `<SSOLoginPage />` directly
