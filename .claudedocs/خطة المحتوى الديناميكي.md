# **🧩 خطة هيكلة المحتوى الديناميكي (Dynamic Content & Block Builder)**

**الهدف:** تحويل منصة "بيت سمار" إلى محرك ديناميكي (Headless CMS-style). بدلاً من الحقول النصية الجامدة، سنبني نظام "مكعبات بناء" (Content Blocks) يعتمد على JSONB، مقسم عبر 3 مستويات هرمية، مع إعطاء الأدمن مرونة في التحكم بالتنسيق (عناوين ذهبية كبيرة، نصوص وصفية، أيقونات).

## **🏛️ 1\. الهيكلة الهرمية للمحتوى (The Tri-Level Hierarchy)**

لفصل المحتوى الذكي وعدم التكرار، سنقسم البيانات إلى 3 مستويات:

1. **المستوى العام (Global Level):** \- *المكان:* جدول Client  
   * *المحتوى:* القواعد العامة للمنتجع (مثل أوقات الهدوء، سياسة الإلغاء العامة، معلومات المواصلات).  
2. **مستوى الفئة (Category Level \- Templates):**  
   * *المكان:* حقل category في جدول Unit (شاليه، فيلا، ستوديو).  
   * *الوظيفة:* يعمل كـ "قالب افتراضي" (Template). إذا اختار الأدمن "فيلا"، يقوم الداشبورد تلقائياً بحقن الـ JSON الخاص بمرافق الفلل (مسبح، حديقة، 3 غرف).  
3. **مستوى الوحدة (Unit Level \- Overrides):**  
   * *المكان:* جدول Unit.  
   * *المحتوى:* الوصف الدقيق، الإطلالة، وأي استثناءات خاصة بهذه الوحدة فقط.

## **🗄️ 2\. تحديث قاعدة البيانات (Addressing DB Report Gaps)**

بناءً على database\_report.md (الفجوات 1, 2, 3)، سنقوم بتحديث نموذج Unit في schema.prisma كالتالي:

* category String (إلزامي: villa, chalet, studio \- للفلترة والقوالب).  
* description\_ar String? / description\_en String? (دعم ثنائية اللغة للوصف الأساسي).  
* content\_blocks Json? (نظام المكعبات الديناميكي للتفاصيل).  
* amenities Json? (مرافق الوحدة مع الأيقونات).  
* rules\_policies Json? (قواعد وسياسات الإلغاء المخصصة للوحدة إن وُجدت).

## **🧱 3\. هندسة مكعبات الـ JSON (The Block Builder Pattern)**

لكي نعطي الأدمن القدرة على تحديد "عناوين بخط كبير ولون ذهبي" و "نصوص صغيرة" كما طلبت، سنبرمج حقل content\_blocks ليكون مصفوفة من الكائنات (Array of Objects) يقرأها الـ React كالتالي:

\[  
  {  
    "type": "section\_title",  
    "content": "Connection, Nature & Serenity 🌿",  
    "style": { "size": "large", "color": "gold", "bold": true }  
  },  
  {  
    "type": "highlight\_item",  
    "icon": "Sparkles",  
    "title": "Luxurious Accommodation",  
    "content": "King-size beds with fine linens..."  
  },  
  {  
    "type": "paragraph",  
    "content": "Discover Zayin Patio Cottage at Beit Smar...",  
    "style": { "size": "normal", "color": "gray" }  
  }  
\]

**في الواجهة الأمامية (Frontend):**

مكون \<DynamicContentRenderer blocks={unit.content\_blocks} /\> سيقرأ هذا الـ JSON:

* إذا كان type \=== "section\_title" → يعرض \<h2\> بـ Tailwind classes (مثال: text-2xl text-\[\#d4a853\] font-black).  
* إذا كان type \=== "highlight\_item" → يعرض أيقونة من lucide-react بجانب عنوان ونص.

## **⚙️ 4\. واجهة لوحة التحكم (Admin UI Builder)**

في UnitFormModal.jsx (أو صفحة تعديل الوحدة المتقدمة)، لن يتعامل الأدمن مع كود الـ JSON أبداً.

سنبني له واجهة **(Builder UI)**:

* ➕ **إضافة عنوان رئيسي** (يفتح حقل نص \+ اختيار اللون: ذهبي/أسود).  
* ➕ **إضافة ميزة بارزة (Highlight)** (يفتح حقل لاسم الأيقونة، العنوان، والوصف).  
* ➕ **إضافة نص عادي** (مربع نص Textarea).

*عند الحفظ، تقوم واجهة React بتحويل هذه المدخلات إلى مصفوفة الـ JSON الموضحة أعلاه وإرسالها للباك إند.*
