### [SEC-01] 🔴 Plain-text Password Comparison في `security.py`

**الملف:** `app/core/security.py`

```python
# ❌ خطر جداً — إذا كان الباسورد غير مشفر، بيقارنه مباشرة!
def verify_password(plain_password: str, hashed_password: str) -> bool:
    if not hashed_password.startswith("$2"):
        return plain_password == hashed_password  # ← TIMING ATTACK + plain text
```

**المشكلة:**
- أي باسورد غير مشفر بـ bcrypt بيُقبل كـ plain text
- يسمح بـ timing attack (المهاجم يعرف إذا الباسورد صح أم لا بناءً على وقت الاستجابة)
- إذا اخترق أحد الـ database، يشوف الباسوردات مباشرة

**الإصلاح:**
```python
def verify_password(plain_password: str, hashed_password: str) -> bool:
    # دائماً استخدم bcrypt — لا استثناءات
    if not hashed_password.startswith("$2"):
        logger.error("⚠️ Plain-text password detected in DB — run scripts/fix_passwords.py immediately")
        return False  # ← ارفض plain text، لا تقارنه
    try:
        return bcrypt.checkpw(
            plain_password.encode('utf-8'),
            hashed_password.encode('utf-8')
        )
    except ValueError:
        return False
```

### [SEC-04] 🟡 Multi-Tenant Phone Uniqueness في `Customer`

**الملف:** `prisma/schema.prisma`

```prisma
model Customer {
  phone String @unique  // ← unique globally عبر كل التناتس!
  email String? @unique // ← نفس المشكلة
}
```

**المشكلة:** إذا عندك tenant A و tenant B، وعميل بنفس رقم الهاتف حجز على الاثنين، الـ `create` برح يفشل بـ unique constraint error.

**الإصلاح:**
```prisma
model Customer {
  phone    String
  clientId String @map("client_id") @db.Uuid
  // الـ unique constraint يكون مركّب على الاثنين معاً
  @@unique([clientId, phone])
  // وأيضاً:
  @@unique([clientId, email])
}
```

    return this.props.children;

}
